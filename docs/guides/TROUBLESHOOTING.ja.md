# 🔧 トラブルシューティング

> 🌐 **Languages**: [English](TROUBLESHOOTING.md) | [日本語](TROUBLESHOOTING.ja.md) | [中文](TROUBLESHOOTING.zh.md) | [한국어](TROUBLESHOOTING.ko.md) | [Español](TROUBLESHOOTING.es.md)

SRT Probe の使用中に問題が発生した場合の解決方法をまとめています。

## 接続できない

**症状**: テストを開始しても「Disconnected」のまま

**解決方法**:
1. ファイアウォールで SRT Probe が許可されているか確認
2. ポート番号が正しいか確認（送信側と受信側で同じポート番号を使用）
3. 片方が Listener モードで待機しているか確認
4. IP アドレスが正しいか確認

## 統計が表示されない

**症状**: 接続は成功するが、RTT やスループットが表示されない

**解決方法**:
1. データが実際に送受信されているか確認
2. Sender モードで起動しているか確認（Receiver のみでは統計が表示されない場合があります）
3. ログ出力を確認してエラーがないかチェック

## 言語が変更されない

**症状**: 言語を選択しても UI が変わらない

**解決方法**:
1. アプリを一度終了して再起動
2. 設定ファイルを確認: `C:\Users\[ユーザー名]\.srt-probe\settings.json`
3. 設定ファイルが破損している場合は削除して再起動

## アプリが起動しない

**症状**: アイコンをダブルクリックしても起動しない

**解決方法**:
1. Windows のバージョンを確認（Windows 10/11 が必要）
2. Microsoft Visual C++ 再頒布可能パッケージがインストールされているか確認
3. イベントビューアーでエラーログを確認

## よくある質問（FAQ）

### Q1: 言語を変更したのに反映されません
**A:** アプリを一度完全に終了して、再起動してください。設定ファイルが読み込まれます。

### Q2: ファイアウォールの設定方法がわかりません
**A:** Windows Defender ファイアウォールで、SRT Probe.exe を許可リストに追加してください。

**手順:**
1. Windows セキュリティを開く
2. 「ファイアウォールとネットワーク保護」を選択
3. 「ファイアウォールによるアプリケーションの許可」をクリック
4. 「SRT Probe」を探して、プライベートとパブリックの両方にチェックを入れる

### Q3: RTTが常に0msと表示されます
**A:** データがまだ送受信されていない可能性があります。Sender モードで接続しているか確認してください。

### Q4: プリセットが保存されません
**A:** 設定ファイルの保存先フォルダに書き込み権限があるか確認してください。通常は `C:\Users\[ユーザー名]\.srt-probe\` です。

### Q5: 接続できません（Disconnected のまま）
**A:** 以下を確認してください：
- 相手側が先に起動しているか（Listener モードの場合）
- IPアドレスとポート番号が正しいか
- ファイアウォールでポートが開放されているか
- ネットワークが接続されているか

## その他の問題

上記で解決しない場合は、[GitHub Issues](https://github.com/videosupporter/srt-probe/issues) で問題を報告してください。
